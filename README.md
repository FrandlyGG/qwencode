# 🤖 Telegram Bot для мониторинга слотов Wildberries

Бот автоматически отслеживает доступные слоты для отгрузки на складах Wildberries и отправляет уведомления в Telegram.

## ✨ Возможности

- **Автоматический мониторинг** - периодическая проверка складов WB
- **Фильтр по городу** - выбирайте только нужные вам города
- **Мгновенные уведомления** - бот сразу сообщает о новых слотах
- **Гибкая настройка** - интервал проверки, выбор конкретных складов
- **Статистика работы** - отслеживание количества проверок и уведомлений
- **Логирование** - подробные логи для отладки

## 📋 Требования

- Python 3.10+
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))
- Wildberries API Token (из личного кабинета селлера)
- Chat ID для уведомлений

## 🚀 Установка

### 1. Клонируйте репозиторий или скачайте файлы

```bash
cd /path/to/project
```

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Настройте переменные окружения

Скопируйте файл `.env.example` в `.env` и заполните его:

```bash
cp .env.example .env
```

Откройте `.env` и заполните необходимые значения:

```env
# Токен API Wildberries
WB_API_TOKEN=your_wildberries_api_token_here

# Токен Telegram бота
TG_BOT_TOKEN=your_telegram_bot_token_here

# ID чата для уведомлений
TG_CHAT_ID=your_chat_id_here

# Интервал проверки (секунды)
CHECK_INTERVAL=60

# ID складов для мониторинга (опционально)
MONITOR_WAREHOUSE_IDS=

# Уровень логирования
LOG_LEVEL=INFO

# Файл логов
LOG_FILE=wb_monitor.log
```

### 4. Как узнать Chat ID?

1. Добавьте бота в чат (или начните диалог)
2. Отправьте любое сообщение
3. Перейдите по ссылке: `https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates`
4. Найдите `"chat":{"id": ...}` в ответе

Или используйте бота [@userinfobot](https://t.me/userinfobot)

## 🔧 Настройка

### Фильтр по городу

Используйте команду `/city` чтобы указать город для поиска складов.
Бот будет показывать только склады в указанном городе.

Примеры:
- `Москва`
- `Санкт-Петербург`
- `Казань`

Для снятия фильтра отправьте пустое сообщение после команды `/city`.

### Мониторинг конкретных складов

В файле `.env` укажите ID складов через запятую:

```env
MONITOR_WAREHOUSE_IDS=123,456,789
```

Оставьте пустым для мониторинга всех складов.

### Интервал проверки

Измените интервал проверки в секундах:

```env
CHECK_INTERVAL=30  # Проверка каждые 30 секунд
```

## 📝 Логи

Все события записываются в файл `wb_monitor.log` и выводятся в консоль.

Уровни логирования:
- `DEBUG` - подробная отладочная информация
- `INFO` - основные события (по умолчанию)
- `WARNING` - предупреждения
- `ERROR` - ошибки

## ⚠️ Troubleshooting

### Ошибка "chat not found"

Убедитесь, что:
1. Бот добавлен в чат или у него есть личные сообщения с вами
2. Chat ID указан правильно в `.env`
3. Бот имеет права на отправку сообщений в чат

### Бот не находит слоты

1. Проверьте правильность токена WB API
2. Убедитесь, что выбранный город имеет доступные склады
3. Попробуйте снять фильтр города
4. Проверьте логи на наличие ошибок

### Rate limit ошибки

API Wildberries ограничивает количество запросов. 
Увеличьте интервал проверки (`CHECK_INTERVAL`) если получаете много ошибок 429.
