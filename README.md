# 🤖 Wildberries Slot Monitor Bot

Бот для мониторинга свободных слотов на складах Wildberries с фильтрацией по городу.

## 🚀 Быстрый старт

### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Настройка конфигурации

Создайте файл `.env` в папке с ботом:

```env
# Токен бота от @BotFather
TG_BOT_TOKEN=your_bot_token_here

# Токен API Wildberries (получить в личном кабинете селлера)
WB_API_TOKEN=your_wb_api_token_here

# ID чата для уведомлений (можно оставить пустым)
TG_CHAT_ID=

# Уровень логирования: DEBUG, INFO, WARNING, ERROR
LOG_LEVEL=DEBUG
```

### 3. Запуск бота
```bash
python wb_monitor_bot.py
```

## 📋 Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Главное меню с кнопками |
| `/start_monitoring` | Запустить мониторинг |
| `/stop_monitoring` | Остановить мониторинг |
| `/status` | Показать статистику |
| `/city` | Установить город для фильтрации |
| `/help` | Справка по боту |

## 🌍 Как выбрать город

1. Нажмите кнопку **"🌍 Выбрать город"** или введите `/city`
2. Бот покажет текущий город и попросит ввести новый
3. Введите название города (например: `Москва`, `Санкт-Петербург`, `Казань`)
4. Бот будет показывать только склады в этом городе
5. Чтобы сбросить фильтр, отправьте **пустое сообщение**

## 🔧 Основные возможности

✅ **Фильтрация по городу** - показывает только нужные вам склады  
✅ **Автоматический мониторинг** - проверка каждые 30 секунд  
✅ **Уведомления** - мгновенное оповещение о новых слотах  
✅ **Статистика** - количество проверок, находок и ошибок  
✅ **Подробное логирование** - все действия записываются в `wb_monitor.log`  
✅ **Защита от ошибок** - бот не падает при проблемах с API

## 🔍 Отладка проблем

### Бот не отправляет сообщения
1. Проверьте токен бота в `.env`
2. Убедитесь, что вы написали боту `/start`
3. Проверьте логи на наличие ошибок

### Бот не принимает город
1. Нажмите `/start` чтобы вернуть главное меню
2. Нажмите **"🌍 Выбрать город"** или введите команду `/city`
3. Введите название города **текстовым сообщением** (например: `Москва`)
4. После отправки город сохранится, и клавиатура вернется

### Клавиатура пропадает при вводе
- Это нормальное поведение Telegram при вводе текста
- После отправки сообщения клавиатура **автоматически вернется**
- Если не вернулась - нажмите `/start` для возврата в главное меню

### Ошибка "chat not found"
- Бот теперь отправляет сообщения тому, кто запустил мониторинг
- Убедитесь, что вы написали боту хотя бы одно сообщение

## 🆘 Troubleshooting

**Проблема:** Бот пишет "Получено 128 складов", но ничего не показывает  
**Решение:** Теперь бот проверяет **все склады**, а не только первые 10. Установите город через `/city` для фильтрации

**Проблема:** Ошибка `TypeError: unsupported operand type(s) for |: 'Command' and 'str'`  
**Решение:** Исправлено в новой версии! Теперь используется два отдельных декоратора `@router.message` вместо объединения через `|`

**Проблема:** Unclosed client session  
**Решение:** Нормально при остановке бота, исправлено в новой версии

**Проблема:** Rate limit  
**Решение:** Бот автоматически ждет 60 секунд при получении rate limit от API
